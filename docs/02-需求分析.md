# 智学伴 - 需求分析

## 一、用户体系需求

### 1.1 登录方式
| 项目 | 需求 |
|------|------|
| 登录方式 | 邮箱+密码登录 |
| 注册方式 | 邮箱注册 |
| 密码要求 | 至少6位 |

### 1.2 用户信息
| 项目 | 需求 |
|------|------|
| 基础信息 | 邮箱、密码、昵称、头像 |
| 学习目标 | 每日目标时长、每周目标时长、每月目标时长 |
| 个人偏好 | 深色模式、通知开关 |

### 1.3 数据安全
| 项目 | 需求 |
|------|------|
| 密码加密 | 使用BCrypt加密存储 |
| 数据注销 | 支持用户注销删除所有数据 |

---

## 二、科目模块需求

### 2.1 预设科目
| 科目 | 说明 |
|------|------|
| 行测 | 公务员行政职业能力测验 |
| 申论 | 公务员申论考试 |
| 公共基础知识 | 事业编公共基础知识 |
| 其他 | 用户可自定义 |

### 2.2 科目管理
| 项目 | 需求 |
|------|------|
| 自定义科目 | 用户可以自己创建科目 |
| 科目颜色 | 不同科目用不同颜色区分 |
| 科目图标 | 每个科目一个图标 |
| 科目排序 | 用户可以调整显示顺序 |
| 科目数量 | 不限制，建议不超过10个 |

---

## 三、番茄钟模块需求

### 3.1 计时功能
| 项目 | 需求 |
|------|------|
| 计时方式 | 正计时 |
| 显示信息 | 倒计时+科目 |
| 时间单位 | 分钟 |

### 3.2 计时控制
| 项目 | 需求 |
|------|------|
| 开始计时 | 选择科目后开始计时 |
| 暂停计时 | 能暂停，但要记录原因 |
| 恢复计时 | 暂停后可以恢复 |
| 停止计时 | 结束计时，记录学习记录 |

### 3.3 专注度判断
| 项目 | 需求 |
|------|------|
| AI判断 | 根据中断次数判断专注度 |
| 用户自评 | 用户自己打分（1-5分） |

### 3.4 暂停惩罚
| 项目 | 需求 |
|------|------|
| 暂停原因 | 必须记录暂停原因 |
| AI判断 | AI判断是否需要惩罚 |

### 3.5 休息提醒
| 项目 | 需求 |
|------|------|
| 休息提醒 | 计时结束后提醒用户休息 |
| 自动休息 | 不自动进入休息模式 |

---

## 四、学习记录模块需求

### 4.1 记录内容
| 字段 | 说明 |
|------|------|
| 科目 | 学习的科目 |
| 时长 | 学习时长（分钟） |
| 时间段 | 开始时间-结束时间 |
| 备注 | AI自动生成 |
| 心情 | 用户自定义选择 |
| 专注度 | 用户自评+AI判断 |

### 4.2 记录规则
| 项目 | 需求 |
|------|------|
| 排序方式 | 按科目分组 |
| 筛选方式 | 按科目筛选 |
| 搜索功能 | 不需要 |
| 删除功能 | 不能删除 |
| 修改功能 | AI判断给不给修改机会 |
| 完整性 | 保证数据真实性，防止作弊 |

### 4.3 记录展示
| 项目 | 需求 |
|------|------|
| 时间格式 | 小时+分钟（如2小时30分钟） |
| 显示方式 | 时间段+时长 |

---

## 五、打卡模块需求

### 5.1 打卡规则
| 项目 | 需求 |
|------|------|
| 打卡时间 | 自然日（晚上12点前打卡算当天） |
| 打卡条件 | 学习完+写完日记才能打卡成功 |
| 连续打卡 | 记录连续打卡天数 |

### 5.2 断签处理
| 项目 | 需求 |
|------|------|
| 断签记录 | 记录断签原因 |
| 断签惩罚 | 只记录不惩罚，周报月报提醒用户 |
| 补签机制 | AI判断给不给补签 |

### 5.3 打卡展示
| 项目 | 需求 |
|------|------|
| 今日状态 | 显示今日是否打卡 |
| 连续天数 | 显示连续打卡天数 |
| 历史记录 | 查看打卡历史 |

## 七、日记模块需求（核心功能）

### 7.1 打卡关系
| 项目 | 需求 |
|------|------|
| 强制性 | 每天必须写日记才能打卡成功 |
| 流程 | 学习完→写日记→打卡成功 |
| 补签日记 | 补签时写原因也算写日记 |

### 7.2 日记内容
| 字段 | 说明 |
|------|------|
| 今日总结 | 今天干了什么 |
| 明日计划 | 明天打算干什么 |
| 心得体会 | 学习感悟 |
| 问题记录 | 遇到什么问题 |
| 内容格式 | Markdown |
| 字数要求 | 不限字数 |

### 7.3 日记图片
| 项目 | 需求 |
|------|------|
| 数量限制 | 不限数量 |
| 大小限制 | 单张不超过10MB |
| 图片格式 | JPG/PNG/GIF |
| 存储路径 | 按用户ID分目录 |
| 图片展示 | 缩略图，点击放大 |

### 7.4 AI日记助手
| 项目 | 需求 |
|------|------|
| AI生成内容 | 基于今日学习记录+心情+专注度 |
| AI生成方式 | 用户提供关键词或简单描述，AI扩展 |
| AI生成风格 | 简洁客观+鼓励性+分析性 |
| AI生成长度 | 用户自定义 |
| AI生成时机 | 用户点击生成按钮 |
| AI生成次数 | 每天最多3次 |
| AI生成修改 | 用户修改后还能重新生成 |
| AI生成预览 | 先预览再保存 |
| AI生成提示 | 要提示用户AI正在生成 |
| AI生成失败 | 提示用户可以自己写，服务恢复后继续使用 |

### 7.5 日记查看
| 项目 | 需求 |
|------|------|
| 查看方式 | 日历视图，像朋友圈一样 |
| 查看内容 | 可以查看之前的日记 |
| 搜索功能 | 不需要 |

### 7.6 日记隐私
| 项目 | 需求 |
|------|------|
| 可见范围 | 只有自己能看 |
| 社交功能 | 没有 |

### 7.7 日记管理
| 项目 | 需求 |
|------|------|
| 保存方式 | 手动保存 |
| 编辑功能 | 不能编辑 |
| 删除功能 | 不能删除 |
| 编辑历史 | 不需要记录 |
| 统计信息 | 简单统计（篇数等） |

### 7.8 日记提醒
| 项目 | 需求 |
|------|------|
| 提醒功能 | 不需要提醒（因为是强制的） |

### 7.9 日记导出
| 项目 | 需求 |
|------|------|
| 导出功能 | 不需要 |
